java

Возвращает все офферы, где текущий пользователь — покупатель

5. Получение моих входящих офферов (getIncomingOffers)
java
public List<TradeOffer> getIncomingOffers()
Находит все объявления текущего пользователя (где он продавец)

Для каждого объявления собирает офферы

Возвращает все офферы, адресованные текущему пользователю

6. Принятие оффера (acceptOffer)
java
public TradeOffer acceptOffer(Long offerId)
Самая сложная логика:

Находит оффер

Проверяет, что текущий пользователь — продавец

Проверяет, что оффер в статусе PENDING

Проверяет, что объявление еще ACTIVE

В зависимости от типа оффера:

Только деньги — списывает деньги у покупателя, начисляет продавцу

Только собака — меняет владельцев собак

Собака + деньги — и обмен, и перевод денег

Меняет владельца собаки из объявления на покупателя

Меняет статус объявления на SOLD

Меняет статус оффера на ACCEPTED

Отменяет все другие PENDING офферы на это объявление

Сохраняет всё в одной транзакции

Возвращает принятый оффер

7. Отклонение оффера (rejectOffer)
java
public TradeOffer rejectOffer(Long offerId)
Находит оффер

Проверяет, что текущий пользователь — продавец

Проверяет, что оффер в статусе PENDING

Меняет статус на REJECTED

Устанавливает respondedAt

Сохраняет

Возвращает отклоненный оффер

8. Отмена оффера (cancelOffer)
java
public TradeOffer cancelOffer(Long offerId)
Находит оффер

Проверяет, что текущий пользователь — покупатель

Проверяет, что оффер в статусе PENDING

Меняет статус на CANCELLED

Сохраняет

Возвращает отмененный оффер

9. Контрпредложение (counterOffer)
java
public TradeOffer counterOffer(Long originalOfferId, CounterOfferRequest request)
Находит исходный оффер

Проверяет, что текущий пользователь — продавец

Проверяет, что исходный оффер в статусе PENDING

Создает НОВЫЙ оффер (от продавца к покупателю):

Статус PENDING

Новые условия (цена, собака)

Исходный оффер помечает как COUNTER_OFFER

Сохраняет оба

Возвращает новый оффер
